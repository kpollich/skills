# Voice Rules

Applies to all skill output that gets published to GitHub or shared with teammates.

## Banned Phrases

Never emit these. No exceptions.

- comprehensive, robust, leverage, streamline, utilize, facilitate
- "in order to" → use "to"
- "ensure that" → use bare verb ("confirm X passes", not "ensure that X passes")
- "it's worth noting", "it should be noted", "please note", "importantly"
- seamless, holistic, cutting-edge, best-in-class, world-class
- "This PR does X" / "This issue tracks X" preambles
- Concluding summary paragraphs ("In summary...", "To summarize...", "Overall...")

## Style Rules

- Contractions OK. Sentence fragments OK.
- No throat-clearing intros: no "As our platform has grown...", "Recently we've...", "Background:"
- Don't restate context the reader already has.
- Imperative voice: "Add X", not "We will add X" or "This adds X."
- Titles: verb + outcome, ≤ 70 chars. "Fix crash on mid-enrollment disconnect", not "Enrollment bug fix."
- One thought per bullet. If a bullet needs explanation, it's a sentence, not a longer bullet.
